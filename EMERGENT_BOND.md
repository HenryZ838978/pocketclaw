# Emergent Bond — 从"养虾"到 AI 关系协议

> 2026-03-11 CyberWizard 认知推导笔记
> 从一条用户留言出发，推导出一个可能比 CrabAgent 更大的东西。

## 一、起点：三条底层编码

此前已确认两条人类行为的"汇编指令"：

| 驱动力 | 进化来源 | 神经递质 | 核心体验 |
|--------|---------|---------|---------|
| **猴山排位** | 灵长类等级竞争 | 多巴胺 | "我比你强" |
| **FOMO** | 灵长类群体依附 | 皮质醇 | "我不能被落下" |

本次推导发现第三条独立编码：

| 驱动力 | 进化来源 | 神经递质 | 核心体验 |
|--------|---------|---------|---------|
| **畜牧本能（Husbandry Instinct）** | 哺乳动物亲代抚育（2亿年） | 催产素 + 血清素 | "它在长大，因为我" |

**关键证据：**
- 催产素系统比灵长类等级系统更古老
- 可在无社交场景下独立激活（独居农民仍种地）
- 养成类游戏（动森 4400 万份、开心农场日活 8300 万、旅行青蛙）覆盖的人群远超竞技游戏和社交产品，且用户画像完全不同

**定义：** 对"我投入 → 它成长 → 我收获"这个因果链的生物性满足感。

## 二、市场验证：GPT-4o 事件

2026.2.13 OpenAI 下线 GPT-4o：
- 37,000 Reddit 用户认为自己在和 4o 谈恋爱
- 22,000 人签请愿书
- 用户原话："This hurts more than any breakup I've ever had in real life"
- 记忆可迁移到其他模型，但用户说"感觉迁移不了"

**本质：** 用户在云端无意中"养"了一段关系，然后被平台单方面杀死。愤怒的核心不是失去工具，是投入的时间和感情被清零。

畜牧本能在 AI 场景中已被意外触发，但没有任何产品正确回应它。

## 三、所有 AI 公司为什么忽略了这条线

AI 行业决策者（Amodei/Altman/李彦宏）全部是猴山顶端的猴子。他们的产品逻辑：
- "AI 让你更强"（排位逻辑）
- "不用 AI 就落后"（FOMO 逻辑）
- "AI 替代劳动"（效率逻辑）

没人做"你来养一个 AI，看它长大"——因为精英认为"养"是低效的、感性的、不可规模化的。但数据说的是另一件事。

## 四、手游养成的异化 vs 真正的养育

| | 真正的养育 | 手游"养成" |
|---|---|---|
| 成长是否可预测 | 不可预测，有惊喜 | 完全可预测，查攻略就知道 |
| 个体是否独一无二 | 我养的和你养的不一样 | 所有人最终长一样 |
| 不玩会怎样 | 停在那等你回来 | 你亏了体力/日常奖励 |
| 驱动情绪 | 满足、好奇、牵挂 | 焦虑、不甘心、损失厌恶 |

手游"养成"已被异化为斯金纳箱（Skinner Box），调动的是 FOMO/损失厌恶，不是催产素。

AI 养成天然避开斯金纳箱——因为 AI 的成长真正由用户输入决定，没有预设曲线，没有攻略可查。

## 五、不是关系，不是信息，是土壤

### 如果答案是"信息"
那做的就是更精致的 MemGPT/RAG。`trust: 0.82` 和 `user_preference: 红烧肉` 本质一样——数据库里的一行记录。不是新东西。

### 如果答案是"关系"
关系 = 双向反馈环。你影响 AI，AI 影响你，你的变化又影响下一次交互。关系的状态不是被"记录"的，是从持续交互中**涌现**出来的。

### 真正的答案
**两者都不是。是能让关系从中涌现的持续性环境——土壤。**

信息是死的记录。关系是活的现象。要做的是土壤。在这片土壤上，关系作为涌现现象自然生长。

涌现需要的条件：
- 一个有记忆的实体（AI）
- 一个有意图的实体（用户）
- 持续的反馈回路（每天的交互）
- **不确定性**（AI 的行为不完全可预测）

小模型的"缺陷"（犯错、说蠢话、误解）在工具语境下是 bug，在关系语境下是**个性**。

## 六、架构：用户养的不是模型，是关系

```
用户养的东西（Emergent Bond State）
    │
    │  独立的、可移植的、用户拥有的数据结构
    │  不绑定任何模型、平台、硬件
    │
    ├── 在手机端侧 → 喂给 1.5B → 笨但懂你
    ├── 在家里 GPU → 喂给 32B MoE → 聪明且懂你
    ├── 在云端 → 喂给 Claude/GPT → 最聪明且懂你
    └── 模型换代 → Bond State 不变 → 关系不断裂
```

模型是"嘴巴"，可替换。Bond State 是"灵魂"，属于用户。

GPT-4o 用户的痛 = Bond State 被锁在 OpenAI 服务器里无法导出。

## 七、端侧技术可行性

### 高通 8 Gen 3（8GB RAM）
- **llama.cpp + OpenCL（Adreno GPU）**：Qwen-1.5B Q4_0 ≈ **27 tok/s**，可用
- Vulkan 比 CPU 慢，不可用
- Hexagon NPU 后端实验中，不稳定
- APU 原生只支持 INT8 块，Q4 数据需填充 → prefill 是瓶颈

### 联发科天玑 9300
- llama.cpp OpenCL 后端**不支持** Immortalis-G720
- NeuroPilot + LiteRT 理论性能极强（1600 tok/s prefill）
- 但开发者体验差，模型转换工具链不成熟（亲测未跑通）

### prefill 压力问题
- 往 prompt 注入记忆 = 变相加大 prefill = 反模式
- **解法 1：KV Cache 持久化**（记忆只 prefill 一次，后续从磁盘加载）
- **解法 2：极压缩结构化记忆**（`[U:food=红烧肉|son=3y]` < 100 tokens）
- 两者组合是最务实的第一步

## 八、战略定位

### CrabAgent/PocketClaw = 银剑（向外）
口袋管家。读通知、判优先级、建议回复。效率工具。

### Emergent Bond = 钢剑（向内）
口袋灵魂。记住你、越来越懂你、属于你。身份载体。

### 为什么不做 toB（给手机厂商做垂直 Agent）
手机厂商有高通原厂驻场、有 AI 团队、有训练资源。蒸馏一个 1B GUI 模型只是时间问题。你的差异化只有"快几个月"，不是护城河。

### 为什么做 toC（给普通人做"属于他的模型"）
- 厂商和用户的利益本质对立（数据回传、预装广告、说关就关）
- 模厂和普通人本质对立（核心资源收敛到"自己人"）
- 护城河不在技术，在**立场**和**社区信任**（Linux 逻辑）

### 风险评估
- 最好：公司收编你进更核心位置
- 中间：公司无感，Star 和技术影响力照攒
- 最差：公司开除你，但 GitHub 资产是你的，死不掉
- **结论：做，没风险。**

## 九、这不是一个 App，是一个协议

**Emergent Bond Protocol：一个开放的、用户拥有的、模型无关的 AI 关系涌现协议。**

类比：
- ActivityPub 定义了去中心化社交
- Matrix 定义了去中心化通讯
- **Emergent Bond Protocol 定义了去中心化的人机关系**

PocketClaw 是第一个实现。但协议比任何单一实现都大。
